# Fork Workspace

Status: Active

## Purpose

This fork exists to support custom product, design, and implementation changes while staying maintainable alongside upstream updates. This workspace stores fork-owned strategy, design, engineering, and upstream-sync context. Upstream contributor guidance and shared project docs continue to live elsewhere under `docs/`.

## Directory Map

- `docs/fork/strategy/`: Durable fork direction, goals, and upstream-alignment rules.
- `docs/fork/design/`: Product, UX, and customization notes.
- `docs/fork/engineering/`: Implementation backlog, technical notes, and implementation plans.
- `docs/fork/upstream/`: Merge/rebase context and upstream sync history.

## How To Use These Docs

- Capture durable fork direction, goals, and alignment rules in `docs/fork/strategy/`.
- Capture new product ideas in `docs/fork/design/` and technical ideas in `docs/fork/engineering/`.
- Promote approved work into focused design specs or implementation plans when ready to guide real work.
- Record upstream merge/rebase context in `docs/fork/upstream/upstream-sync-log.md`.
