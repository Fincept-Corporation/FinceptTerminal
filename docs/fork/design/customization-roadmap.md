# Customization Roadmap

Status: Active

## Priorities

- Define the fork's own product direction and usage workflow.
- Plan design changes that make the interface fit the fork's needs.
- Track enabling technical work needed to keep approved customizations manageable during upstream updates.

## User-Facing Changes

- Adjust branding, terminology, and navigation to reflect the fork's purpose.
- Refine or redesign selected screens and workflows once priorities are approved.

## Notes

- Start with documentation and planning so later code changes stay traceable.
- Prefer isolated customizations that are easier to preserve across upstream syncs.
